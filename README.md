## Introduction

IT Support is essential for maintaining the stability, security, and efficiency of an organization's technology environment. It involves managing service requests, resolving incidents, troubleshooting technical issues, and supporting end users across various systems and devices.

This repository serves as a knowledge base for IT Service Operations, documenting support processes, best practices, and operational references. Its purpose is to promote consistent service delivery, improve response times, and support continuous improvement in IT operations while ensuring a reliable and productive experience for users.


## Objective

To analyze IT support operations and service requests to identify trends, improve response efficiency, enhance service quality, and support informed decision-making.

## Dataset Information

The dataset contains IT service requests and incident records, including request details, support activities, completion times, vendor involvement, service categories, and customer feedback ratings.

### Key Fields
- Request Date
- Requester Name
- Department
- Title (Hardware / Software)
- Status Progress
- Assignees
- Hours Attend
- Days to Complete
- Vendor Information
- Rating

## Key Performance Indicators (KPIs)

The dashboard tracks several operational metrics:

- Total Requests
- Completed Requests
- Open Requests
- Average Resolution Time
- Average Days to Complete
- Vendor Involvement Rate
- Average User Rating
- Support Workload by Assignee

## Key Insights

- Hardware-related requests generated a higher number of tickets compared to software requests.
- Most requests were resolved within the targeted service period.
- Certain departments generated significantly more support tickets than others.
- Vendor-assisted requests required longer resolution times compared to internally resolved requests.
- User satisfaction remained high based on average ratings received after ticket closure.
